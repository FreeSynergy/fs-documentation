## Übersicht

FreeSynergy nutzt ein zweigeteiltes i18n-System:

1. **Eingebettete Snippets** (`fsn-i18n` Lib) — EN und DE fest einkompiliert, immer verfügbar
2. **Store-Sprachpakete** — alle weiteren 48 Sprachen, zur Laufzeit aus dem Store ladbar

Dateiformat: **TOML** für Snippets. Schlüssel-Konvention: `"kategorie.schlüssel"`.

---

## Snippet-Kategorien

Jede Sprache hat 12 TOML-Dateien:

| Datei | Inhalt | Beispiele |
|---|---|---|
| `actions.toml` | Verben / Aktionen | `save`, `cancel`, `delete`, `deploy` |
| `nouns.toml` | Substantive | `host`, `project`, `service`, `database` |
| `status.toml` | Statusmeldungen | `running`, `stopped`, `healthy`, `error` |
| `errors.toml` | Fehlermeldungen | `not_found`, `network_error`, `permission_denied` |
| `validation.toml` | Formular-Validierung | `required`, `too_short`, `invalid_email` |
| `phrases.toml` | Ganze Sätze / Phrasen | `confirm_delete`, `save_success` |
| `time.toml` | Zeitangaben | `now`, `never`, `ago`, `in_seconds` |
| `help.toml` | Hilfe-Texte | Tastatur-Shortcuts, UI-Erklärungen |
| `labels.toml` | UI-Labels | Formular-Beschriftungen |
| `confirmations.toml` | Bestätigungs-Dialoge | `really_delete`, `discard_changes` |
| `notifications.toml` | Benachrichtigungen | `update_available`, `deploy_done` |
| `meta.toml` | Sprach-Metadaten | Eigenname, Sprachfamilie |

Gesamt: ~350–400 Basis-Snippets pro Sprache.

## Einbettung (Compile-Time)

Nur **EN und DE** sind mit `include_str!` fest einkompiliert — sie sind immer verfügbar, ohne Store:

```rust
// fsn-i18n/src/snippets.rs
snippet!("en", "actions.toml"),
snippet!("en", "errors.toml"),
// ...
snippet!("de", "actions.toml"),
// ...
```

Alle anderen 48 Sprachen kommen aus dem Store.

## Variablen in Snippets

```toml
[errors]
file_not_found    = "File not found: { $path }"
service_down      = "Cannot reach service { $name } at { $url }"
permission_denied = "Permission denied: { $action } on { $resource }"
```

## Fallback-Kette

```
Angefragte Sprache (z.B. DE)
  → Nicht gefunden?
    → Fallback auf EN
      → Nicht gefunden?
        → Key als Text anzeigen
```

## Store-Sprachpakete

Jede Sprache ist ein vollständiges Store-Paket vom Typ `language` unter `shared/i18n/{code}/`.

### Paket-Inhalt

```
shared/i18n/de/
  manifest.toml         ← Paket-Metadaten
  icon.svg              ← Sprachcode-Icon (Sprechblase, 24×24)
  actions.toml
  nouns.toml
  status.toml
  errors.toml
  validation.toml
  phrases.toml
  time.toml
  help.toml
  labels.toml
  confirmations.toml
  notifications.toml
  meta.toml
```

### Manifest-Struktur

```toml
[meta]
id            = "lang-de"
name          = "Deutsch"
resource_type = "language"
description   = "German (Germany) language pack"
version       = "1.1.0"
author        = "FreeSynergy"
license       = "MIT"
icon          = "icon.svg"
tags          = ["german", "deutsch", "de", "language"]

[language]
locale       = "de-DE"
display_name = "Deutsch"
direction    = "ltr"
completeness = 100
programs     = ["node", "desktop", "init"]
family       = "Indo-European"
continent    = "Europe"
```

### Filter-Eigenschaften

`family` und `continent` sind **eigene Pflichtfelder** (keine Tags) — so können sie beim Erstellen nicht vergessen werden, und der Store/Validator kann sie prüfen.

| Feld | Typ | Beispiele |
|---|---|---|
| `family` | Sprachfamilie | `Indo-European`, `Afro-Asiatic`, `Sino-Tibetan`, `Dravidian`, `Uralic`, `Turkic`, `Austronesian`, `Niger-Congo`, `Japonic`, `Koreanic`, `Kra-Dai`, `Austroasiatic` |
| `continent` | Kontinent | `Europe`, `Asia`, `Africa`, `Americas`, `Oceania` |

## LanguageMeta (Rust-Struct)

In `fsn-i18n/src/languages.rs`, geladen aus `languages.toml`:

```rust
pub struct LanguageMeta {
    pub code:        &'static str,  // "de", "ar", "yue"
    pub name:        &'static str,  // "German"
    pub native_name: &'static str,  // "Deutsch"
    pub script:      &'static str,  // "Latin", "Cyrillic", "Arabic", ...
    pub direction:   TextDirection, // Ltr / Rtl
    pub family:      &'static str,  // "Indo-European"
    pub continent:   &'static str,  // "Europe"
}
```

Zugriff:
```rust
use fsn_i18n::language_meta;

let m = language_meta("de").unwrap();
// m.name == "German"
// m.family == "Indo-European"
// m.continent == "Europe"
// m.is_rtl() == false
```

## RTL/LTR

4 RTL-Sprachen: `ar` (Arabisch), `fa` (Persisch), `ur` (Urdu), `ps` (Pashto).

CSS nutzt **logische Properties** für RTL-Kompatibilität:
- `margin-inline-start` statt `margin-left`
- `padding-inline-end` statt `padding-right`
- `inset-inline-start` statt `left`

## API-Übersicht (`fsn-i18n`)

```rust
// Initialisieren
fsn_i18n::init(Path::new("locales")).unwrap();
fsn_i18n::init_with_builtins().unwrap();  // EN+DE direkt ohne Dateien

// Übersetzen
let label = fsn_i18n::t("actions.save");
let msg   = fsn_i18n::t_with("errors.file_not_found", &[("path", "/etc/foo")]);

// Sprach-Metadaten
let m = language_meta("ja").unwrap();
// m.family == "Japonic"  |  m.continent == "Asia"

// Alle Sprachen durchlaufen
for lang in all_languages() { ... }
```
