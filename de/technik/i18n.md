## Technologie

Mozilla Fluent (.ftl Dateien) über das `fluent` Crate.

## Snippet-Kategorien

| Datei | Inhalt | Beispiel-Anzahl |
|---|---|---|
| `actions.ftl` | Verben/Aktionen | ~40 |
| `nouns.ftl` | Substantive | ~30 |
| `status.ftl` | Statusmeldungen | ~25 |
| `errors.ftl` | Fehlermeldungen | ~60 |
| `validation.ftl` | Formular-Validierung | ~30 |
| `phrases.ftl` | Ganze Sätze | ~40 |
| `time.ftl` | Zeitangaben | ~25 |
| `help.ftl` | Hilfe-Texte | ~30 |
| `labels.ftl` | UI-Labels | ~40 |
| `confirmations.ftl` | Bestätigungs-Dialoge | ~15 |
| `notifications.ftl` | Benachrichtigungen | ~20 |

Gesamt: ~350-400 Basis-Snippets.

## Fehler-Snippets mit Variablen

```fluent
error-file-not-found = File not found: { $path }
error-service-unreachable = Cannot reach service { $name } at { $url }
error-permission-denied = Permission denied: { $action } on { $resource }
```

## Fallback-Kette

```
Angefragte Sprache (z.B. DE)
  → Nicht gefunden?
    → Fallback auf EN
      → Nicht gefunden?
        → Key als Text anzeigen
```

## RTL/LTR

In `languages.toml`:
```toml
[de]
name = "German"           # Immer Englisch
native_name = "Deutsch"   # In eigener Sprache
direction = "ltr"

[ar]
name = "Arabic"
native_name = "العربية"
direction = "rtl"
```

CSS nutzt **logische Properties** für RTL-Kompatibilität:
- `margin-inline-start` statt `margin-left`
- `padding-inline-end` statt `padding-right`
- `inset-inline-start` statt `left`

## Store-Integration

Sprach-Pakete werden über den [Store](../programme/store/README.md) verteilt. Nur installierte Sprachen erscheinen im Dropdown. Englisch ist immer vorinstalliert.

Wenn ein Snippet fehlt → Zuerst Store fragen ob es das gibt, bevor eine Fehlermeldung angezeigt wird.
